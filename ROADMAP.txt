1. Connected Account Flag System ‚úÖ DESIGNED - READY FOR IMPLEMENTATION
   Flag Categories:
   - Account doesn't belong to this user
   - Account is fabricated/doesn't exist
   - Account exists but not connected to user
   - Misleading professional account
   
   Flag Submission Requirements:
   - Category selection (required)
   - Evidence commentary (required)
   - Actual profile URL for verification (required)
   - Consequences explanation shown before submission
   
   Daily Flag Limits (Trust Score Based):
   - 90-100: 5 flags per day
   - 70-89: 3 flags per day
   - 50-69: 2 flags per day
   - Below 50: 1 flag per day
   
   False Flag Consequences (Progressive):
   - 1st offense: Warning + 5 point trust score reduction
   - 2nd offense: Warning + 10 point trust score reduction
   - 3rd offense: Warning + 15 point trust score reduction
   - Continued false flagging: Escalating trust score drops
   - Low trust score (below 50): Restricted from commenting/posting about others
   - Groups with trust score minimums become inaccessible
   - Profile remains unaffected, only trust score drops
   
   Confirmed Fake Account Consequences:
   - 2-3 day restriction from posting/commenting about others
   - 10-20 point trust score reduction (progressive for repeat offenses)
   - Temporary loss of connected account privileges (1 week)
   - Flagged accounts hidden until resolution
   
   Moderation System:
   - Timeline: 1-3 days for flag resolution (depends on moderator availability)
   - Moderator tools: View flagged profile, compare friends lists, verify evidence
   - Flagged accounts hidden immediately until resolved
   - Anonymous flagging (flagging user identity protected)
   - Inbox notification to flagged user (anonymous)
   - Dispute process: User can provide proof of ownership
   - Appeals system for false flag claims

2. Moderator/Admin Interface for Flag Management ‚úÖ DESIGNED - READY FOR IMPLEMENTATION
   Interface Design:
   - Email inbox-style layout with flag packages
   - Each package contains comprehensive situation information
   - Decision buttons at bottom with dialogue box for explanation
   
   Information Package Contents:
   - Flagged user's full profile + connected accounts
   - Flagging user's full profile + identity (for credibility assessment)
   - Evidence provided (category, commentary, actual profile URL)
   - Comparison data (mutual friends, account creation dates)
   - Flag history for both users (past flags made/received)
   - Trust scores of both users
   
   Decision Options:
   - Approve Flag (apply consequences to flagged user)
   - Deny Flag (apply false flag consequences to flagging user)
   - Need More Info (request additional evidence)
   - Custom explanation required for all decisions (no templates)
